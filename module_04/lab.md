## Задание 1. Анализ длины строковых полей [DAX]

**Бизнес-задача:** Администратор БД хочет оценить заполненность текстовых полей справочника оборудования.

**Требования:**
- Выведите: название оборудования, длину названия, длину инвентарного номера, длину модели, длину описания производителя
- Добавьте столбец `total_text_length` — суммарная длина всех строковых полей
- Отсортируйте по убыванию суммарной длины

**Подсказка SQL:** `LENGTH()`, `COALESCE()` для обработки NULL.

**Ожидаемый результат:** таблица со всеми 18 единицами оборудования.

<details>
<summary>Решение (DAX)</summary>

```dax
EVALUATE
    ADDCOLUMNS(
        dim_equipment,
        "name_len", LEN(dim_equipment[equipment_name]),
        "inv_len", LEN(dim_equipment[inventory_number]),
        "model_len", LEN(dim_equipment[model]),
        "manuf_len", LEN(dim_equipment[manufacturer]),
        "total_text_length",
            LEN(dim_equipment[equipment_name])
            + LEN(dim_equipment[inventory_number])
            + LEN(dim_equipment[model])
            + LEN(dim_equipment[manufacturer])
    )
ORDER BY [total_text_length] DESC
```

</details>

## Задание 2. Разбор инвентарного номера

**Бизнес-задача:** Нужно автоматически классифицировать оборудование по инвентарному номеру.

**Требования:**
- Из инвентарного номера формата `INV-LHD-001` извлеките:
  - Префикс (INV)
  - Код типа (LHD, TRUCK, CART, SKIP)
  - Серийный номер (001) — как целое число (без ведущих нулей)
- Добавьте столбец с расшифровкой типа:
  - LHD → «Погрузочно-доставочная машина»
  - TRUCK → «Шахтный самосвал»
  - CART → «Вагонетка»
  - SKIP → «Скиповой подъёмник»
- Отсортируйте по коду типа и серийному номеру

**Подсказка SQL:** `SPLIT_PART()`, `CAST()`, `CASE WHEN`.

## Задание 3. Формирование краткого имени оператора [DAX]

**Бизнес-задача:** Для отчётов нужны краткие имена операторов в формате «Фамилия И.О.».

**Требования:**
- Сформируйте краткое имя: `Иванов И.П.` (учтите, что `middle_name` может быть NULL)
- Сформируйте имя в формате: `И.П. Иванов` (инициалы перед фамилией)
- Выведите фамилию в верхнем регистре
- Выведите должность в нижнем регистре
- Отсортируйте по фамилии

**Ожидаемый результат:** все операторы из таблицы `dim_operator`.

<details>
<summary>Решение (DAX)</summary>

```dax
EVALUATE
    ADDCOLUMNS(
        dim_operator,
        "short_name_1",
            dim_operator[full_name],
        "upper_last",
            UPPER(LEFT(dim_operator[full_name],
                FIND(" ", dim_operator[full_name]) - 1)),
        "lower_position",
            LOWER(dim_operator[position])
    )
ORDER BY dim_operator[full_name]
```

**Примечание:** В SSAS-модели поле `full_name` уже содержит объединённое ФИО оператора. Для формирования кратких имён в формате «Фамилия И.О.» в DAX можно использовать комбинацию `LEFT`, `MID` и `FIND`.

</details>

## Задание 4. Поиск оборудования по шаблону

**Бизнес-задача:** Инженер хочет быстро находить оборудование по части названия или характеристике.

**Требования:**
- Найдите оборудование, в названии которого есть слово «ПДМ» (с помощью LIKE)
- Найдите оборудование производителей, начинающихся на «S» (латинская) — без учёта регистра
- Найдите все шахты, в описании названия которых есть кавычки (`"`)
- С помощью регулярного выражения найдите инвентарные номера, серийная часть которых содержит только цифры от 001 до 010

**Подсказка SQL:** `LIKE`, `ILIKE`, `POSITION`, `~` (POSIX regex).

## Задание 6. Возраст оборудования [DAX]

**Бизнес-задача:** Главный механик хочет знать возраст каждой единицы оборудования для планирования замены.

**Требования:**
- Вычислите полный возраст оборудования от даты ввода в эксплуатацию (используя `AGE`)
- Выведите возраст в годах (целое число)
- Выведите количество дней в эксплуатации
- Классифицируйте:
  - Менее 2 лет → «Новое»
  - 2-4 года → «Рабочее»
  - Более 4 лет → «Требует внимания»
- Отсортируйте по убыванию возраста

**Ожидаемый формат:**

| Оборудование | Дата ввода | Возраст | Лет | Дней | Категория |
|---|---|---|---|---|---|
| ПДМ-01 | 2020-03-15 | 5 years 11 mons | 5 | 2194 | Требует внимания |

<details>
<summary>Решение (DAX)</summary>

```dax
EVALUATE
    ADDCOLUMNS(
        dim_equipment,
        "commissioning_date", dim_equipment[commissioning_date],
        "years",
            DATEDIFF(
                dim_equipment[commissioning_date],
                TODAY(),
                YEAR
            ),
        "days",
            DATEDIFF(
                dim_equipment[commissioning_date],
                TODAY(),
                DAY
            ),
        "category",
            SWITCH(
                TRUE(),
                DATEDIFF(dim_equipment[commissioning_date], TODAY(), YEAR) < 2, "Новое",
                DATEDIFF(dim_equipment[commissioning_date], TODAY(), YEAR) <= 4, "Рабочее",
                "Требует внимания"
            )
    )
ORDER BY [days] DESC
```

</details>

## Задание 8. Анализ простоев по дням недели и часам [DAX]

**Бизнес-задача:** Определить, в какие дни недели и часы чаще всего происходят простои.

**Требования:**
- Извлеките из `start_time` (TIMESTAMP): день недели, час, минуту
- Сгруппируйте простои по дням недели — подсчитайте количество и среднюю длительность
- Сгруппируйте простои по часам — подсчитайте количество
- Определите «пиковый час» — час с наибольшим количеством простоев
- Используйте `DATE_TRUNC('hour', start_time)` для группировки по часам

**Ожидаемый формат (по дням недели):**

| День недели | Количество | Средняя длительность (мин) |
|---|---|---|
| Понедельник | 12 | 45.3 |
| Вторник | 8 | 52.1 |
| ... | ... | ... |

**Подсказка SQL:** `EXTRACT(DOW FROM ...)`, `EXTRACT(HOUR FROM ...)`, `AVG()`, `COUNT()`.

<details>
<summary>Решение (DAX)</summary>

```dax
// По дням недели
EVALUATE
    SUMMARIZECOLUMNS(
        dim_date[day_of_week_name],
        "downtime_count", COUNTROWS(fact_equipment_downtime),
        "avg_duration", AVERAGE(fact_equipment_downtime[duration_min])
    )
ORDER BY [downtime_count] DESC
```

**Примечание:** В SSAS-модели `dim_date[day_of_week_name]` содержит название дня недели. Для группировки по часам можно использовать `dim_time[hour]`, если факт простоев связан с измерением времени. Если нет — DAX не имеет прямого аналога `EXTRACT(HOUR FROM timestamp)`, поэтому для анализа по часам рекомендуется использовать SQL.

</details>

## Задание 9. Расчёт графика калибровки датчиков

**Бизнес-задача:** Определить, какие датчики требуют калибровки (калибровка должна проводиться каждые 180 дней).

**Требования:**
- Для каждого датчика рассчитайте:
  - Количество дней с последней калибровки
  - Дату следующей плановой калибровки (`calibration_date + 180 дней`)
  - Статус: «Просрочена» (более 180 дней), «Скоро» (150-180 дней), «В норме» (менее 150 дней)
- Присоедините название оборудования и тип датчика
- Отсортируйте: сначала просроченные, затем по дате калибровки

**Подсказка SQL:** `AGE()`, `INTERVAL`, `CASE WHEN`, несколько `JOIN`.

## Задание 10. Комплексный отчёт: карточка оборудования

**Бизнес-задача:** Сформировать текстовую «карточку» каждой единицы оборудования для системы мониторинга.

**Требования:**
- Сформируйте одну строку-описание для каждой единицы оборудования в формате:

```
[ПДМ] ПДМ-01 (Sandvik LH517i) | Шахта: Северная | Введён: 15.03.2020 | Возраст: 5 лет | Статус: АКТИВЕН | Видеорег.: ДА | Навигация: ДА
```

- Требования к формату:
  - Тип оборудования в квадратных скобках
  - Производитель и модель в скобках
  - Дата в формате `ДД.ММ.ГГГГ`
  - Возраст — только полные годы
  - Статус — в верхнем регистре и на русском (active → АКТИВЕН, maintenance → НА ТО, decommissioned → СПИСАН)
  - Видеорег. и Навигация: ДА/НЕТ

**Подсказка SQL:** `CONCAT`, `TO_CHAR`, `EXTRACT`, `AGE`, `CASE WHEN`, `UPPER`, `JOIN` с `dim_equipment_type`.
