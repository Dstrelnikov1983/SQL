PostgreSQL не имеет оператора PIVOT. Используем условную агрегацию (CASE WHEN + SUM/COUNT) или функцию crosstab() из расширения tablefunc.

PIVOT через CASE WHEN + агрегат

Добыча руды по шахтам за каждый месяц Q1 2025

-- PIVOT: ежемесячная добыча по шахтам
SELECT
    m.mine_name                                        AS "Шахта",
    SUM(CASE WHEN d.month = 1 THEN p.tons_mined END)  AS "Январь",
    SUM(CASE WHEN d.month = 2 THEN p.tons_mined END)  AS "Февраль",
    SUM(CASE WHEN d.month = 3 THEN p.tons_mined END)  AS "Март",
    SUM(p.tons_mined)                                  AS "Итого Q1"
FROM fact_production p
JOIN dim_date d  ON p.date_id = d.date_id
JOIN dim_mine m  ON p.mine_id = m.mine_id
WHERE d.year = 2025 AND d.quarter = 1
GROUP BY m.mine_name
ORDER BY m.mine_name;
Принцип: каждый CASE WHEN создаёт «виртуальный столбец». Агрегатная функция (SUM, AVG, COUNT) свёртывает значения.

Альтернатива: FILTER (PostgreSQL)

Тот же результат, но более чистый синтаксис

-- PIVOT с использованием FILTER (специфика PostgreSQL)
SELECT
    m.mine_name                                              AS "Шахта",
    SUM(p.tons_mined) FILTER (WHERE d.month = 1)            AS "Январь",
    SUM(p.tons_mined) FILTER (WHERE d.month = 2)            AS "Февраль",
    SUM(p.tons_mined) FILTER (WHERE d.month = 3)            AS "Март",
    SUM(p.tons_mined)                                        AS "Итого Q1"
FROM fact_production p
JOIN dim_date d  ON p.date_id = d.date_id
JOIN dim_mine m  ON p.mine_id = m.mine_id
WHERE d.year = 2025 AND d.quarter = 1
GROUP BY m.mine_name
ORDER BY m.mine_name;
FILTER — расширение PostgreSQL (стандарт SQL:2003). Более читабелен, чем CASE WHEN. Работает с любыми агрегатами: SUM, COUNT, AVG, ARRAY_AGG и др.

Пример «Руда+»: матрица эффективности смен

-- Средняя добыча по типу оборудования и смене
SELECT
    et.type_name                                              AS "Тип оборудования",
    ROUND(AVG(p.tons_mined)
        FILTER (WHERE s.shift_code = 'SHIFT_1'), 1)          AS "Смена 1 (ср.)",
    ROUND(AVG(p.tons_mined)
        FILTER (WHERE s.shift_code = 'SHIFT_2'), 1)          AS "Смена 2 (ср.)",
    ROUND(AVG(p.tons_mined)
        FILTER (WHERE s.shift_code = 'SHIFT_3'), 1)          AS "Смена 3 (ср.)",
    ROUND(AVG(p.tons_mined), 1)                              AS "Общее среднее"
FROM fact_production p
JOIN dim_shift s           ON p.shift_id = s.shift_id
JOIN dim_equipment e       ON p.equipment_id = e.equipment_id
JOIN dim_equipment_type et ON e.equipment_type_id = et.equipment_type_id
JOIN dim_date d            ON p.date_id = d.date_id
WHERE d.year = 2025
GROUP BY et.type_name
ORDER BY et.type_name;

Тема 2. crosstab() из tablefunc

Расширение PostgreSQL для табличного поворота

-- Установка расширения (один раз)
CREATE EXTENSION IF NOT EXISTS tablefunc;

-- crosstab() принимает SQL-запрос, возвращающий 3 столбца:
--   1) идентификатор строки (row_name)
--   2) категория (category) — станет именем столбца
--   3) значение (value)
Формат вызова:
SELECT * FROM crosstab('запрос_данных', 'запрос_категорий') AS ct(столбцы...)

crosstab(): добыча по шахтам и месяцам

SELECT *
FROM crosstab(
    $$
    SELECT m.mine_name,
           d.month_name_short,
           SUM(p.tons_mined)::NUMERIC(12,2)
    FROM fact_production p
    JOIN dim_date d  ON p.date_id = d.date_id
    JOIN dim_mine m  ON p.mine_id = m.mine_id
    WHERE d.year = 2025 AND d.quarter = 1
    GROUP BY m.mine_name, d.month_name_short, d.month
    ORDER BY m.mine_name, d.month
    $$,
    $$ VALUES ('Янв'), ('Фев'), ('Мар') $$
) AS ct (
    "Шахта"   TEXT,
    "Январь"  NUMERIC(12,2),
    "Февраль" NUMERIC(12,2),
    "Март"    NUMERIC(12,2)
);
Особенность: необходимо заранее указать имена и типы столбцов результата. Динамический PIVOT требует PL/pgSQL (составление запроса в runtime).

CASE WHEN vs crosstab()

Критерий CASE WHEN + FILTER crosstab()
Читаемость Высокая, стандартный SQL Средняя, требует $$ и AS
Гибкость Произвольные условия Только «строка-категория-значение»
Производительность Один проход по данным Один проход (оптимизирован)
Динамичность Столбцы фиксированы Столбцы фиксированы
NULL-обработка Явная (COALESCE) Автоматическая
Рекомендация: для аналитических отчётов используйте CASE WHEN / FILTER — проще, понятнее, не требует расширений.

UNPIVOT через UNION ALL

-- Классический подход: UNION ALL для каждого столбца
SELECT sample_number, 'Fe'    AS parameter, fe_content    AS value
FROM fact_ore_quality
WHERE date_id BETWEEN 20250101 AND 20250131
UNION ALL
SELECT sample_number, 'SiO2'  AS parameter, sio2_content  AS value
FROM fact_ore_quality
WHERE date_id BETWEEN 20250101 AND 20250131
UNION ALL
SELECT sample_number, 'Al2O3' AS parameter, al2o3_content AS value
FROM fact_ore_quality
WHERE date_id BETWEEN 20250101 AND 20250131
ORDER BY sample_number, parameter;
Недостаток: таблица сканируется N раз (по числу столбцов). Для больших таблиц — медленно.

UNPIVOT через VALUES + LATERAL

Эффективный способ — один проход по таблице

-- LATERAL + VALUES: один проход по таблице
SELECT
    q.sample_number,
    params.parameter,
    params.value
FROM fact_ore_quality q
CROSS JOIN LATERAL (
    VALUES
        ('Fe',    q.fe_content),
        ('SiO2',  q.sio2_content),
        ('Al2O3', q.al2o3_content),
        ('Влажность', q.moisture)
) AS params(parameter, value)
WHERE q.date_id BETWEEN 20250101 AND 20250131
  AND params.value IS NOT NULL
ORDER BY q.sample_number, params.parameter;
Преимущество LATERAL: таблица сканируется один раз, строки генерируются «на лету». Можно фильтровать NULL-значения сразу.

Тема 4. GROUPING SETS

Несколько уровней группировки в одном запросе

Зачем нужны GROUPING SETS?

Представьте: руководство «Руда+» хочет отчёт, который содержит:

  • Добычу по шахтам и сменам
  • Итого по каждой шахте
  • Итого по каждой смене
  • Общий итог

Без GROUPING SETS нужны 4 отдельных запроса + UNION ALL.

Без GROUPING SETS: 4 запроса + UNION ALL

-- Вариант БЕЗ Grouping Sets — громоздко!
SELECT m.mine_name, s.shift_name, SUM(p.tons_mined)
FROM fact_production p
JOIN dim_mine m  ON p.mine_id = m.mine_id
JOIN dim_shift s ON p.shift_id = s.shift_id
WHERE p.date_id BETWEEN 20250101 AND 20250331
GROUP BY m.mine_name, s.shift_name
UNION ALL
SELECT m.mine_name, 'ИТОГО', SUM(p.tons_mined)
FROM fact_production p
JOIN dim_mine m ON p.mine_id = m.mine_id
WHERE p.date_id BETWEEN 20250101 AND 20250331
GROUP BY m.mine_name
UNION ALL
SELECT 'ИТОГО', s.shift_name, SUM(p.tons_mined)
FROM fact_production p
JOIN dim_shift s ON p.shift_id = s.shift_id
WHERE p.date_id BETWEEN 20250101 AND 20250331
GROUP BY s.shift_name
UNION ALL
SELECT 'ИТОГО', 'ИТОГО', SUM(p.tons_mined)
FROM fact_production p
WHERE p.date_id BETWEEN 20250101 AND 20250331;
Таблица сканируется 4 раза. При больших объёмах — неэффективно.

С GROUPING SETS: один запрос

-- Тот же результат — ОДИН проход по данным
SELECT
    m.mine_name                        AS "Шахта",
    s.shift_name                       AS "Смена",
    SUM(p.tons_mined)                  AS "Добыча, т",
    COUNT(*)                           AS "Записей"
FROM fact_production p
JOIN dim_mine  m ON p.mine_id  = m.mine_id
JOIN dim_shift s ON p.shift_id = s.shift_id
JOIN dim_date  d ON p.date_id  = d.date_id
WHERE d.year = 2025 AND d.quarter = 1
GROUP BY GROUPING SETS (
    (m.mine_name, s.shift_name),  -- детализация: шахта + смена
    (m.mine_name),                -- итого по шахте
    (s.shift_name),               -- итого по смене
    ()                            -- общий итог
)
ORDER BY
    GROUPING(m.mine_name),
    m.mine_name NULLS LAST,
    GROUPING(s.shift_name),
    s.shift_name NULLS LAST;
Один проход! PostgreSQL выполняет агрегацию сразу для всех наборов группировки.

Тема 5. ROLLUP и CUBE

Сокращённая запись для типичных наборов группировки

ROLLUP(A, B)

Иерархическое свёртывание:

  • (A, B) — детализация
  • (A) — итого по A
  • () — общий итог

= GROUPING SETS ((A,B), (A), ())

CUBE(A, B)

Все комбинации:

  • (A, B) — детализация
  • (A) — итого по A
  • (B) — итого по B
  • () — общий итог

= GROUPING SETS ((A,B), (A), (B), ())

ROLLUP: иерархические итоги добычи

-- Иерархия: Шахта -> Ствол -> Смена
-- ROLLUP даёт итоги на каждом уровне иерархии
SELECT
    m.mine_name                AS "Шахта",
    sh.shaft_name              AS "Ствол",
    s.shift_name               AS "Смена",
    SUM(p.tons_mined)          AS "Добыча, т",
    ROUND(AVG(p.tons_mined),1) AS "Ср. добыча"
FROM fact_production p
JOIN dim_mine  m  ON p.mine_id  = m.mine_id
JOIN dim_shaft sh ON p.shaft_id = sh.shaft_id
JOIN dim_shift s  ON p.shift_id = s.shift_id
JOIN dim_date  d  ON p.date_id  = d.date_id
WHERE d.year = 2025 AND d.quarter = 1
GROUP BY ROLLUP (m.mine_name, sh.shaft_name, s.shift_name)
ORDER BY
    m.mine_name  NULLS LAST,
    sh.shaft_name NULLS LAST,
    s.shift_name  NULLS LAST;
ROLLUP идеально подходит для иерархий: регион → город → объект, год → квартал → месяц и т.д.

CUBE: все комбинации для анализа

-- CUBE по двум измерениям: шахта и тип оборудования
-- Получаем ВСЕ комбинации итогов
SELECT
    m.mine_name                 AS "Шахта",
    et.type_name                AS "Тип оборудования",
    SUM(p.tons_mined)           AS "Добыча, т",
    COUNT(DISTINCT p.equipment_id) AS "Ед. техники",
    ROUND(AVG(p.operating_hours), 1) AS "Ср. часы"
FROM fact_production p
JOIN dim_mine m            ON p.mine_id = m.mine_id
JOIN dim_equipment e       ON p.equipment_id = e.equipment_id
JOIN dim_equipment_type et ON e.equipment_type_id = et.equipment_type_id
JOIN dim_date d            ON p.date_id = d.date_id
WHERE d.year = 2025 AND d.quarter = 1
GROUP BY CUBE (m.mine_name, et.type_name)
ORDER BY
    m.mine_name  NULLS LAST,
    et.type_name NULLS LAST;
CUBE с N столбцами генерирует 2^N наборов группировки. С 3 столбцами — 8 комбинаций!

Тема 6. Функция GROUPING()

Отличаем итоговые NULL от «настоящих» NULL

GROUPING(столбец)

  • Возвращает 1 — если столбец «свёрнут» (итоговый NULL)
  • Возвращает 0 — если столбец участвует в группировке

GROUPING() в действии

SELECT
    CASE WHEN GROUPING(m.mine_name)  = 1
         THEN '*** ВСЕ ШАХТЫ ***'
         ELSE m.mine_name END              AS "Шахта",
    CASE WHEN GROUPING(s.shift_name) = 1
         THEN '*** ВСЕ СМЕНЫ ***'
         ELSE s.shift_name END             AS "Смена",
    SUM(p.tons_mined)                      AS "Добыча, т",
    -- Битовая маска для программной обработки
    GROUPING(m.mine_name, s.shift_name)    AS "Уровень"
FROM fact_production p
JOIN dim_mine  m ON p.mine_id  = m.mine_id
JOIN dim_shift s ON p.shift_id = s.shift_id
JOIN dim_date  d ON p.date_id  = d.date_id
WHERE d.year = 2025 AND d.quarter = 1
GROUP BY CUBE (m.mine_name, s.shift_name)
ORDER BY
    GROUPING(m.mine_name),
    m.mine_name NULLS LAST,
    GROUPING(s.shift_name),
    s.shift_name NULLS LAST;

Битовая маска GROUPING()

GROUPING(mine, shift) Двоичное Значение
0 00 Детализация (шахта + смена)
1 01 Итого по шахте (смена свёрнута)
2 10 Итого по смене (шахта свёрнута)
3 11 Общий итог (оба свёрнуты)
Совет: используйте GROUPING() в ORDER BY для красивой иерархической сортировки итогов.

Тема 7. DAX-эквиваленты

Как Power BI решает те же задачи?

SQL (PostgreSQL)

  • PIVOT = CASE WHEN + SUM
  • UNPIVOT = LATERAL + VALUES
  • GROUPING SETS / CUBE / ROLLUP
  • GROUPING() для итогов

DAX (Power BI)

  • PIVOT = Матрица (визуал)
  • UNPIVOT = Power Query Unpivot
  • SUMMARIZECOLUMNS + ROLLUPADDISSUBTOTAL
  • ISSUBTOTAL() для итогов

DAX: PIVOT через матрицу Power BI

SQL (PIVOT)

SELECT mine_name,
  SUM(tons) FILTER
    (WHERE month = 1) AS "Янв",
  SUM(tons) FILTER
    (WHERE month = 2) AS "Фев"
FROM production
GROUP BY mine_name;

DAX

// В Power BI: визуал "Матрица"
// Строки: mine_name
// Столбцы: month_name
// Значения: SUM(tons_mined)

// Или в DAX Studio:
EVALUATE
SUMMARIZECOLUMNS (
    dim_mine[mine_name],
    dim_date[month_name],
    "Добыча", SUM(fact_production[tons_mined])
)
Ключевое отличие: в Power BI PIVOT — это задача визуализации (матрица), а не запроса. DAX не «поворачивает» данные — это делает движок визуалов.

DAX: аналог GROUPING SETS

SQL

SELECT mine_name, shift_name,
       SUM(tons_mined)
FROM fact_production p
JOIN dim_mine  m USING(mine_id)
JOIN dim_shift s USING(shift_id)
GROUP BY ROLLUP(
    mine_name, shift_name
);

DAX

EVALUATE
SUMMARIZECOLUMNS (
    ROLLUPADDISSUBTOTAL (
        dim_mine[mine_name],
        "ИтогоШахта",
        dim_shift[shift_name],
        "ИтогоСмена"
    ),
    "Добыча",
        SUM(fact_production[tons_mined])
)
ORDER BY
    [ИтогоШахта], [mine_name],
    [ИтогоСмена], [shift_name]
ROLLUPADDISSUBTOTAL — DAX-эквивалент ROLLUP. ISSUBTOTAL() — аналог GROUPING().

Сравнение: SQL vs DAX

Операция SQL (PostgreSQL) DAX (Power BI)
PIVOT CASE WHEN + SUM / FILTER Визуал «Матрица»
UNPIVOT LATERAL + VALUES Power Query → Unpivot Columns
GROUPING SETS GROUP BY GROUPING SETS(...) SUMMARIZECOLUMNS + ROLLUPADDISSUBTOTAL
ROLLUP GROUP BY ROLLUP(A, B) ROLLUPADDISSUBTOTAL(A, ..., B, ...)
CUBE GROUP BY CUBE(A, B) ROLLUPADDISSUBTOTAL (ограничено)
Итоговые строки GROUPING(col) ISSUBTOTAL(col)

Квиз: проверим понимание

Вопрос 1

Какой оператор PostgreSQL реализует PIVOT?

A) SELECT ... PIVOT (...)
B) crosstab() — и это единственный способ
C) Условная агрегация: SUM(CASE WHEN ... END) или SUM(...) FILTER (WHERE ...)
D) LATERAL JOIN

Ответ: C. PostgreSQL не имеет оператора PIVOT. Используется условная агрегация. crosstab() — альтернатива, но не единственный способ.

Квиз: проверим понимание

Вопрос 2

Чем ROLLUP(A, B, C) отличается от CUBE(A, B, C)?

A) Ничем — это синонимы
B) ROLLUP даёт 4 набора, CUBE — 8
C) ROLLUP — только для числовых столбцов
D) CUBE работает только с 2 столбцами

Ответ: B. ROLLUP(A,B,C) = (A,B,C), (A,B), (A), () — 4 набора. CUBE(A,B,C) = все 2^3 = 8 комбинаций.

Квиз: проверим понимание

Вопрос 3

Что возвращает GROUPING(mine_name) = 1?

A) mine_name участвует в текущей группировке
B) mine_name «свёрнут» — это итоговая строка
C) mine_name содержит NULL в данных
D) Ошибка — GROUPING() не принимает аргументов

Ответ: B. GROUPING() = 1 означает, что столбец свёрнут (агрегирован), это итоговая строка.

Квиз: проверим понимание

Вопрос 4

Какой DAX-аналог функции GROUPING() в SQL?

A) HASONEVALUE()
B) ISFILTERED()
C) ISSUBTOTAL()
D) ISINSCOPE()

Ответ: C. ISSUBTOTAL() в DAX определяет, является ли строка итоговой — прямой аналог GROUPING() в SQL.

DAX: PIVOT/UNPIVOT — задачи визуализации и Power Query. Grouping Sets — SUMMARIZECOLUMNS + ROLLUPADDISSUBTOTAL + ISSUBTOTAL().
